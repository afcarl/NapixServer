Howto write a Napix module with the low level interface¶

The Napix high level howto teaches about writing and setting up a Napix module that get and serialise a list of resources. When the list become too big or is slow or difficult to save, you may choose to get the resources one by one using the low-level interface.

The base manager of Napix does not require any method to be implemented. It proposes five methods that correspond to the HTTP actions:

Each method implemented will enable the HTTP call in the documentation and metadata of the manager. None is required. You may write a manager that can get a resource, modify it but neither delete, nor create, nor list them.

For this howto, we will take the example of a LVM volume manager.

First steps¶

Napix Class¶

The class we will be using is the base manager of Napix. The doctstring is used by napix to document the service.

The next step is to fill the resource_fields attribute with the fields we will export in our service.

class LVMManager( Manager ):
    """
    LVM logical volumes manager
    """

The service will export the name of the logical volume, its size and the group it belongs to.

    resource_fields = {
            'name' : {
                'description' : 'Name of the logical volume',
                'example' : 'lv_home'
                },
            'group' : {
                'description' : 'Name of the volume group',
                'example' : 'vg01'
                },
            'size' : {
                'description' : 'Total size of the logical volume in Mo',
                'example' : 20000
                }
            }

List the resources¶

The list_resource() method return the list of the possible ID.

We choose to use the uuid of the LVM volume as the identifier of the resource.

Note

We use the executor to run the processes. The executor needs to have its control thread running in order to work.

    def list_resource( self):
        #run the command that spit the UUID line by line
        process = executor.create_job(
                ['sudo','lvm','lvs','--nosuffix', '--noheading',
                    '--separator',';','-o','lv_uuid'], discard_output= False)
        #if the command fail it raises an exception
        # that will be caught by napix and will return a 500 error
        if process.wait() != 0:
            raise subprocess.CalledProcessException(
                'The command exited with a non 0 return code'+
                process.stderr.read())

        return filter( bool,
                map( str.strip, process.stdout.read().split('\n')))

Get a resource¶

The get_resource() method is called with the desired id. This id has already been cleaned by the validate_id() method. If there is nothing corresponding to the id, the method must raise a NotFound exception.

    def _lv_list( self):
        process = executor.create_job(
                ['sudo','lvm','lvs','--nosuffix', '--noheading',
                    '--separator',';','-o',
                    'lv_uuid,lv_size,lv_name,vg_name' ], discard_output= False)
        if process.wait() != 0:
            raise subprocess.CalledProcessException(
                'The command exited with a non 0 return code'+
                process.stderr.read())
        for lv in map( str.strip, process.stdout.read().split('\n')):
            if lv:
                yield lv.split(';')
    def get_resource( self, uuid):
        for lv_uuid, lv_size, lv_name, vg_name in self._lv_list() :
            if uuid == lv_uuid:
                return {
                        'name' : lv_name,
                        'size' : float( lv_size.replace(',','.')),
                        'group' : vg_name
                        }
        raise NotFound, uuid

Validate the input¶

The modify and create method get a data_dict argument. It is cleaned by the validate_resource for the whole resource and each field individually by the the validate_resource_* methods.

The arguments given to the modify and create must be cleaned, to check that the value have the right type and do not contain harmful values, such as ‘..’ in a filename, quotes, or n, etc.

More detail on the validation process here.

We write these validation method in order to ensure that the next methods we will write run safely.

    def _vg_list( self):
        process = executor.create_job(['sudo','lvm','vgs','--noheading','-o','vg_name'])
        process.wait()
        return map( str.strip, process.stdout.split('\n'))
    def validate_resource_name( self, name):
        if '/' in name:
            raise ValidationError, 'Name cannot contain /'
        return name

    def validate_resource_group( self, group):
        vg_list = self._vg_list()
        if group not in vg_list:
            raise ValidationError, 'Group must be one of %s' % ','.join(vg_list)
        return group

    def validate_resource_size( self, size):
        try:
            size = float( size)
            if size > 0:
                return size
        except (ValueError, TypeError):
            pass
        raise ValidationError, 'size must be a positive number'

Create a resource¶

The create_resource() method takes the cleaned data_dict as argument and must return the id of the object it created. If there is a conflict with an existing resource, it may throw a Duplicate exception.

    def create_resource( self, data_dict):
        process = executor.create_job( ['sudo', 'lvcreate',
            '--size', '%dm' % data_dict['size'],
            '--name', data_dict['name'],
            data_dict['group']
            ])
        code= process.wait()
        if code != 0:
            raise Exception, 'command finished with code %d' % code

        for lv_uuid, lv_size, lv_name, vg_name in self._lv_list() :
            if lv_name == data_dict['name'] and vg_name == data_dict['group']:
                return lv_uuid

Delete a resource¶

The delete_resource() get the id of the resource to delete.

    def delete_resource( self, uuid):
        lv = self.get_resource( uuid)
        process = executor.create_job( [ 'sudo', 'lvremove',
            '%s/%s'% ( lv['group'], lv['name']) ] )

        code= process.wait()
        if code != 0:
            raise Exception, 'command finished with code %d' % code
